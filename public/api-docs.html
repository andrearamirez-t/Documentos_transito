API Trámites de Tránsito

API REST para el procesamiento automatizado de documentos de tránsito en Colombia. Permite analizar imágenes de documentos con IA (Gemini), validar los datos extraídos y sincronizarlos con sistemas externos de registro vehicular.

📄
Análisis con IA
Extracción automática de datos mediante Gemini AI desde imágenes de documentos.
✅
Validación
Verificación de campos obligatorios según el tipo de documento de tránsito.
🔁
Sincronización
Integración con sistemas externos para registro y consulta de documentos procesados.

URL Base

Todas las rutas de la API son relativas a la URL base del servidor. En entorno de desarrollo local el servidor corre en el puerto 3000 por defecto.

Base URL
http://localhost:3000
Todos los endpoints retornan respuestas en formato JSON. Se recomienda incluir el header Content-Type: application/json en las peticiones con cuerpo.

Códigos de Respuesta HTTP

La API utiliza los códigos de estado HTTP estándar para indicar el resultado de cada operación.

200
OK
Solicitud procesada correctamente. El cuerpo contiene el recurso o resultado.
201
Created
Recurso creado exitosamente. Se retorna el nuevo objeto en el cuerpo.
400
Bad Request
La solicitud contiene parámetros inválidos o mal formados.
404
Not Found
El recurso solicitado no fue encontrado. Verifique el ID proporcionado.
422
Unprocessable Entity
El payload es válido pero faltan campos requeridos según el tipo de documento.
500
Server Error
Error interno del servidor. Revise los logs para más información.
Documentos

Listar Documentos

Retorna la lista de todos los documentos procesados almacenados en el sistema.

GET /api/documents Listar todos los documentos

Devuelve un arreglo con todos los documentos que han sido procesados y almacenados. Cada documento incluye sus metadatos, tipo, datos extraídos y estado de validación.

Respuestas
Código Descripción
200 OK Arreglo de documentos (puede ser vacío [] si no hay documentos).
500 Error interno al consultar el almacenamiento.
Ejemplo de respuesta — 200 OK
[
  {
    "id":           "doc_1718123456789",
    "tramiteId":    "tramite_1718123456000",
    "fileName":      "tramite_juan.pdf",
    "documentType":  "cedula",
    "extractedData": {
      "nombreCompleto":  "ANDREA RAMÍREZ TORRES",
      "numeroDocumento": "1020456789",
      "fechaNacimiento": "1995-08-14",
      "sexo":            "F"
    },
    "confidence":   0.97,
    "summary":      "Cédula de ciudadanía de Andrea Ramírez Torres.",
    "syncStatus":   "pending",
    "scannedAt":    "2026-06-23T14:30:00.000Z",
    "logs":         []
  }
]

Eliminar Documento

Elimina un documento procesado del sistema por su identificador único.

DELETE /api/documents/:id Eliminar por ID

Elimina permanentemente el documento con el ID especificado. Esta operación no se puede deshacer.

Parámetros de Ruta
Nombre Tipo Requerido Descripción
id string requerido Identificador único del documento a eliminar.
Respuestas
Código Descripción
200 OK Documento eliminado exitosamente.
404 Not Found No existe un documento con el ID proporcionado.

Actualizar Datos Extraídos

Actualiza el campo extractedData de un documento almacenado. Usado por el botón "Guardar" del panel de detalle para persistir correcciones manuales en Firestore.

PATCH /api/documents/:id Guardar correcciones manuales

Reemplaza el contenido de extractedData para el documento indicado sin modificar ningún otro campo (syncStatus, logs, tramiteId, etc.). Ideal para corregir datos que Gemini extrajo incorrectamente.

Parámetros de Ruta
Nombre Tipo Requerido Descripción
id string requerido ID del documento a actualizar.
Cuerpo de la Solicitud (JSON)
Campo Tipo Requerido Descripción
extractedData object requerido Objeto completo con los datos corregidos del documento. Sobreescribe el valor anterior.
Respuestas
Código Descripción
200 OK Datos actualizados exitosamente en Firestore.
404 Not Found No existe un documento con el ID proporcionado (solo en entorno local).
500 Error al actualizar en Firestore.
Ejemplo de solicitud — corregir cédula
// PATCH /api/documents/doc_1718123456789
{
  "extractedData": {
    "nombreCompleto":  "ANDREA RAMÍREZ TORRES",
    "numeroDocumento": "1020456789",
    "fechaNacimiento": "1995-08-14",
    "sexo":            "F"
  }
}
Respuesta — 200 OK
{
  "success": true
}
Este endpoint actualiza solo el campo extractedData. El syncStatus no cambia: si el documento ya estaba sincronizado, seguirá marcado como tal. Para volver a sincronizar con los datos corregidos, llame después a POST /api/documents/sync/:id.

Analizar Documento con IA

Envía un documento al motor de análisis de Gemini AI para extracción automática de datos. Acepta tanto imágenes en base64 como claves de documentos de muestra predefinidos.

POST /api/documents/analyze Análisis con Gemini AI

Procesa el documento enviado y retorna el tipo de documento detectado junto con los datos estructurados extraídos. Si se usa sampleKey, se carga un documento de muestra del sistema. Si se envía base64, se procesa la imagen directamente.

Cuerpo de la Solicitud (JSON)
Campo Tipo Requerido Descripción
sampleKey string opcional Clave del documento de muestra a usar. Opciones: sample_cedula, sample_factura, sample_runt, sample_gases, sample_poder.
fileName string opcional Nombre del archivo para identificar el documento en el historial.
base64 string opcional Imagen del documento codificada en Base64. Requerido si no se especifica sampleKey.
mimeType string opcional Tipo MIME de la imagen (ej: image/jpeg, image/png). Requerido junto con base64.
Respuestas
Código Descripción
201 Created Documento analizado y almacenado. Retorna el objeto con datos extraídos.
400 Bad Request No se proporcionó sampleKey ni base64.
500 Error al comunicarse con la API de Gemini o al procesar la imagen.
Ejemplo con sampleKey
{
  "sampleKey": "sample_cedula"
}
Ejemplo con imagen en Base64
{
  "fileName": "runt_vehiculo.jpg",
  "mimeType": "image/jpeg",
  "base64": "/9j/4AAQSkZJRgABAQAAAQABAAD..."
}

Sincronizar Documento

Sincroniza un documento procesado con un sistema externo de registro.

POST /api/documents/sync/:id Enviar al sistema externo

Toma el documento almacenado con el ID indicado, lo valida y lo envía al sistema externo de destino. Si no se especifica targetUrl, se usa la URL de destino configurada por defecto en el servidor.

Parámetros de Ruta
Nombre Tipo Requerido Descripción
id string requerido ID del documento a sincronizar.
Cuerpo de la Solicitud (JSON)
Campo Tipo Requerido Descripción
targetUrl string opcional URL del sistema externo destino. Si se omite, se usa el valor por defecto del servidor.
Respuestas
Código Descripción
200 OK Documento sincronizado correctamente con el sistema externo.
404 Not Found No se encontró el documento con el ID proporcionado.
422 El documento no pasó la validación de campos requeridos.
500 Error al comunicarse con el sistema externo.
Validación

Validar Documento

Verifica que el payload de un documento contiene todos los campos requeridos según su tipo. Útil para validar datos antes de intentar sincronizarlos.

POST /api/validate Verificar campos obligatorios

Recibe un tipo de documento y un objeto con los datos extraídos. Retorna 200 OK si todos los campos requeridos están presentes, o 422 con la lista de campos faltantes si no lo están.

Cuerpo de la Solicitud (JSON)
Campo Tipo Requerido Descripción
documentType string requerido Tipo de documento: cedula, factura, runt, gases, poder.
extractedData object requerido Objeto con los datos extraídos del documento a validar.
Respuestas
Código Descripción
200 OK Todos los campos requeridos están presentes. Documento válido.
422 Faltan campos requeridos. Se retorna el arreglo missingFields.
400 Bad Request Tipo de documento desconocido o payload malformado.
Ejemplo de solicitud — cédula válida
{
  "documentType": "cedula",
  "extractedData": {
    "nombreCompleto":  "CARLOS MARIO GÓMEZ HERRERA",
    "numeroDocumento": "79854123",
    "fechaNacimiento": "1988-03-22",
    "sexo":            "M"
  }
}
Respuesta — 200 OK
{
  "valid": true,
  "documentType": "cedula"
}
Ejemplo de solicitud — campos faltantes
{
  "documentType": "cedula",
  "extractedData": {
    "nombreCompleto": "CARLOS MARIO GÓMEZ HERRERA"
    /* faltan: numeroDocumento, fechaNacimiento, sexo */
  }
}
Respuesta — 422 Unprocessable Entity
{
  "valid":         false,
  "documentType":  "cedula",
  "missingFields": [
    "numeroDocumento",
    "fechaNacimiento",
    "sexo"
  ]
}

Campos Requeridos por Tipo

Cada tipo de documento de tránsito colombiano tiene un conjunto de campos obligatorios que deben estar presentes en extractedData para que la validación sea exitosa.

cedula Cédula de Ciudadanía
  • nombreCompleto
  • numeroDocumento
  • fechaNacimiento
  • sexo
factura Factura de Compraventa
  • numeroFactura
  • fecha
  • compradorNombre
  • compradorIdentificacion
  • vehiculoMarca
  • vehiculoLinea
  • vehiculoModelo
  • vehiculoMotor
  • vehiculoChasis
  • valorTotal
runt RUNT — Registro Único
  • tramiteSolicitado
  • propietarioNombre
  • propietarioIdentificacion
  • vehiculoMarca
  • vehiculoLinea
  • vehiculoMotor
  • vehiculoChasis
gases Cert. de Gases
  • tipoCertificado
  • fechaCertificado
  • vehiculoMarca
  • vehiculoChasis
  • vehiculoMotor
  • vehiculoCilindrada
poder Poder Notarial
  • otorganteNombre
  • otorganteIdentificacion
  • apoderadoNombre
  • tramitesAutorizados
Sistema Externo

Registrar en Sistema Externo

Endpoint simulado del sistema externo de destino. Recibe el payload completo del documento, valida los campos requeridos y lo registra en la base de datos del sistema de destino.

POST /api/external-system-mock Simular recepción en sistema destino

Actúa como el receptor en el sistema de destino. Valida que el payload contenga documentType y extractedData con todos los campos requeridos para el tipo indicado. Si la validación es exitosa, almacena el registro.

Cuerpo de la Solicitud (JSON)
Campo Tipo Requerido Descripción
documentType string requerido Tipo de documento: cedula, factura, runt, gases, poder.
extractedData object requerido Objeto con los datos extraídos del documento. Debe incluir todos los campos requeridos según el tipo.
Respuestas
Código Descripción
201 Created Documento registrado exitosamente en el sistema externo.
422 Campos requeridos faltantes en extractedData.
400 Bad Request El documentType no es reconocido o el payload está malformado.
Ejemplo de solicitud — registro de factura
{
  "documentType": "factura",
  "extractedData": {
    "numeroFactura":            "FCV-2026-00142",
    "fecha":                    "2026-06-15",
    "compradorNombre":          "LUIS FERNANDO OSPINA",
    "compradorIdentificacion":  "8820134",
    "vehiculoMarca":            "CHEVROLET",
    "vehiculoLinea":            "SPARK GT",
    "vehiculoModelo":           2024,
    "vehiculoMotor":            "MTR-4B12-0045678",
    "vehiculoChasis":           "9BWZZZ377VT004251",
    "valorTotal":              58500000
  }
}
Respuesta — 201 Created
{
  "success":    true,
  "registerId": "ext_1750688400123",
  "receivedAt": "2026-06-23T14:40:00.123Z"
}

Consultar Base de Datos Destino

Retorna todos los registros almacenados en la base de datos del sistema externo de destino.

GET /api/external-system-db Obtener registros del destino

Devuelve todos los documentos que han sido registrados exitosamente en el sistema externo mediante el endpoint /api/external-system-mock.

Respuestas
Código Descripción
200 OK Arreglo con todos los registros almacenados (puede ser vacío).
500 Error interno al consultar la base de datos.
Ejemplo de respuesta — 200 OK
[
  {
    "registerId":   "ext_1750688400123",
    "documentType": "factura",
    "extractedData": { /* ... */ },
    "receivedAt":   "2026-06-23T14:40:00.123Z"
  }
]

Limpiar Base de Datos Destino

Elimina todos los registros almacenados en la base de datos del sistema externo.

DELETE /api/external-system-db/clear Vaciar base de datos destino

Borra todos los registros del sistema externo de destino. Útil para reiniciar el estado durante pruebas o en ambientes de desarrollo.

Esta operación es irreversible. Todos los registros del sistema de destino serán eliminados permanentemente.
Respuestas
Código Descripción
200 OK Base de datos vaciada exitosamente.
500 Error interno al intentar limpiar la base de datos.
Respuesta — 200 OK
{
  "cleared": true,
  "deletedCount": 5
}
